How the 16×16 tiled matmul kernel works

A thread still computes one output value. The speedup comes from 256 threads cooperating: they load small pieces of A and B once into shared memory, then reuse those values sixteen times.

C[M×N] = A[M×K] × B[K×N]

A two-dimensional block contains 16 × 16 = 256 threads. Block (blockIdx.x, blockIdx.y) owns one 16×16 region of C, and thread (threadIdx.x, threadIdx.y) owns one element inside it.

3. Cooperatively load shared memory

Each of the 256 threads loads one A value and one B value. Together they fill two complete 16×16 arrays in shared memory.

4. Reuse each loaded value sixteen times

Thread (tx, ty) reads row ty from the shared A tile and column tx from the shared B tile. That creates sixteen contributions to its private accumulator.

5. Handle matrices that are not multiples of 16

The grid uses ceiling division, so it may create blocks that extend beyond a matrix edge. Invalid input positions load zero; invalid output threads still participate in both barriers but skip their final store.

// Partial A or B tile:
a_tile[ty][tx] = valid_a
    ? A[row * K + a_col]
    : 0.0f;

b_tile[ty][tx] = valid_b
    ? B[b_row * N + col]
    : 0.0f;
// Partial C tile:
if (row < M && col < N) {
    C[row * N + col] = total;
}

Why there is no early return: every thread in a block must reach both __syncthreads() barriers. Returning only the out-of-range threads could leave the remaining threads waiting forever.

The complete algorithm

constexpr int tile = 16;

__global__ void matmul_f32(A, B, C, M, K, N) {
    __shared__ float a_tile[tile][tile];
    __shared__ float b_tile[tile][tile];

    row = blockIdx.y * tile + threadIdx.y;
    col = blockIdx.x * tile + threadIdx.x;
    float total = 0;

    for (start = 0; start < K; start += tile) {
        // All threads cooperatively stage the next tiles.
        a_tile[ty][tx] = valid A value or zero;
        b_tile[ty][tx] = valid B value or zero;
        __syncthreads();

        // Each thread updates its one output accumulator.
        for (inner = 0; inner < tile; ++inner)
            total += a_tile[ty][inner] * b_tile[inner][tx];
        __syncthreads();
    }

    if (row < M && col < N)
        C[row * N + col] = total;
}

Production source: src/devices/cuda/cuda_device.cu. This remains a readable reference kernel; cuBLAS and CUTLASS use deeper register tiling, pipelining, architecture-specific instructions, and runtime kernel selection.
